🧠 演算法核心思想

1. 狀態定義

dp(k, i, j) = 有 k 個信箱，需要測試區間 [i, j] 時，最壞情況下需要的最少炸藥數

2. 狀態轉移

對於每個可能的測試點 x (i ≤ x ≤ j)：

dp(k, i, j) = min(x + max(dp(k-1, i, x-1), dp(k, x+1, j))) | | | 花費的炸藥 信箱炸毀情況 信箱未炸毀情況

3. 為什麼用 max？

因為我們要考慮最壞情況！不管信箱炸毀或沒炸毀，我們都要為最糟的情況做準備。

4. 基礎情況

  • i > j：區間為空，答案已確定，返回 0
  • k == 0：沒有信箱了但還沒確定答案，返回無限大（不可行）

💡 解題關鍵洞察

1. 這不是普通的二分搜尋！

雖然看起來像二分搜尋，但關鍵差異是：信箱會被炸毀！所以我們不能總是選擇中間點。

2. 為什麼要考慮最壞情況？

因為我們需要保證在任何情況下都能找到答案。如果我們只考慮平均情況，可能會遇到炸藥不夠的尷尬局面！

3. 狀態空間的含義

  • k：可用的信箱數量（資源限制）
  • [i, j]：可能的承受範圍（搜尋空間）
  • 回傳值：保證找到答案需要的最少炸藥數

4. 記憶化的重要性

同樣的子問題會被重複計算很多次，記憶化可以將時間複雜度從指數級降到多項式級！

時間複雜度: O(k × m³) 空間複雜度: O(k × m²)
